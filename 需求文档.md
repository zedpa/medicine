## B. 本次新增需求（spec-002 · T1）

详见 `specs/spec-002-research-closed-loop.md`。该 spec 同时列出后续切片 T2–T6（韦恩图 PNG、STRING PPI 网络、GO/KEGG 富集、方法学&引用生成、网页可视化），按 P4「一次只推进一个 Stage」逐个过闸。

### 功能需求
- **FR-T1.1** `run_herb(query, disease=None)` / `analyze_herbs(herbs, disease?)` / CLI `--disease`：疾病参数可选，缺省时行为与 v1.0 完全一致（向后兼容）。
- **FR-T1.2** 疾病名→人类相关基因(gene symbol)+关联分数，按 `disease.score_min` 过滤；本地缓存(ns=disease)，可复现。
- **FR-T1.3** `intersect_targets(drug, disease)` 纯函数 → `intersection / drug_only / disease_only / counts`，结构直接喂韦恩图。
- **FR-T1.4** 提供疾病时 Excel 增 `疾病靶点`、`交集靶点` 两 sheet。
- **FR-T1.5** 口径集中于 `config/pipeline.yaml › disease`（source/score_min/max_targets/timeout），改动记 `CHANGELOG.md`。

### 口径（写入 config，遵守 P5）
- 疾病数据源：Open Targets Platform GraphQL（免密钥，0–1 关联强度）
- `disease.score_min = 0.1`（关联强度下限，可调）

### 功能需求（T3 · PPI 网络）

## B2. 本次新增需求（spec-003 · 侧边栏对话历史 · T1）

### 功能需求（T1 · 历史存储层）
- **FR-H1.1** `HistoryStore(db_path)` sqlite 持久化会话：建表幂等、并发安全（沿用 `cache.py` 的 `_LOCK`）。
- **FR-H1.2** `save/get/list/delete/prune`：按 `id` upsert；`get` 缺失返回 `None`；`list` 按 `updated_at`
  降序仅返回元信息（含 `n_messages`，**不含** `messages` 正文）；`delete` 幂等；`prune(keep)` 留最新 N 条。
- **FR-H1.3** `_derive_title(messages, max_len)` 纯函数：取首条 user 消息为标题，截断加省略号，无 user 回退「新对话」。
- **FR-H1.4** 时间戳/id 由调用方注入，存储层不调 `datetime.now()`/`uuid` → 纯净、确定性可单测
  （与 workflow 禁用 `Date.now()` 同理：时间是输入不是副作用）。
- **FR-H1.5** 口径集中于 `config/pipeline.yaml › history`（db_path/max_conversations/title_max_len），改动记 CHANGELOG。

### 口径（写入 config，遵守 P5）
- 新增 `history` 段：`db_path=data/history.sqlite`（独立于 cache.sqlite）、`max_conversations=50`、`title_max_len=24`。
- `.gitignore` 增 `data/*.sqlite`（本地用户数据，不预置会话，P3）。无业务阈值变更（纯持久化层）。

### 验收（T1，先写测试，红-绿）
- AC-1…AC-8 见 spec-003 §4.2，对应 `tests/test_history.py`（标题截断/中文不坏 ensure_ascii=False/
  get 缺失 None/upsert 覆盖不产生两行/降序+limit+n_messages/幂等删/prune 留最新 N/二次实例读落盘）。
- 现状：`pytest tests/test_history.py` 8 passed；全套 **39 passed** 无回归。

### 功能需求（T2 · 侧边栏 UI 接入）
- **FR-H2.1** 侧边栏「💬 对话历史」区：`store.list(limit=max_conversations)` 渲染会话标题(最近在上)，当前会话标 🟢。
- **FR-H2.2** 点击会话 → 载入其 `messages` 到 `session_state` 并切 `conv_id`，`st.rerun()` 重绘。
- **FR-H2.3** 「➕ 新建对话」先落库当前会话再清空并换新 `conv_id`；每条标题旁「🗑」`store.delete`(删当前则清屏)。
- **FR-H2.4** 每轮回复后 `_save_current` 自动落库 + `prune`；「清空对话」改为「清空当前对话内容」(仅清屏不删库)。
- **FR-H2.5** `conv_id`/`created_at`/`updated_at` 由 UI 层(uuid4/datetime)生成后注入存储层 → 存储层保持纯净。
- **口径**：store 用 `@st.cache_resource` 复用 sqlite 连接；`db_path` 可被环境变量 `HISTORY_DB_PATH` 覆盖(供 E2E 隔离)。
