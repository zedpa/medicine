# Spec-008 侧边栏结构重整（对齐 Claude 网页版）

状态: ✅ 完成 · 负责人: web · 关联: [[spec-006 UI 重设计]](spec-006-ui-redesign.md) · [[spec-003 对话历史]](spec-003-conversation-history.md) · 方法: SDD

## 1. 背景与问题（用户反馈 + 实景截图）

spec-003/006 后侧边栏出现结构与排版问题：
- **「⋯」菜单挤成「.·」**：会话行 `st.columns([6,1])` 使菜单列仅 ~14% 宽，popover 触发按钮被压扁。
- **字体不自适应**：侧栏标题沿用宋体（`h3` 全局宋体），偏重；长会话标题**换行**破坏行结构；「直接模式(...)」「管理员」等**折行**。
- **「清空对话」已无意义**：有了持久化历史后，清空当前对话不如直接「新建对话」；且该按钮埋在折叠 expander 里，几乎不可达。
- **整体结构混乱**：账户/后端信息挤在底部 expander 大盒子里。

目标：按 **Claude 网页版**侧栏结构重整——品牌 → 新建对话 → 最近对话（扁平列表）→ 底部账户区——并修复上述排版；移除「清空对话」。

保持功能不破：保留全部 E2E 依赖的控件名/文本（新建对话、最近对话、会话标题按钮、「⋯」+ 删除对话、管理后台、登出、「直接模式」）。

## 2. 变更

- **结构（Claude 式四段）**：
  1. 品牌（紧凑、**无衬线**）；
  2. `✚ 新建对话`（松绿主按钮）；
  3. `最近对话` 扁平列表——标题**幽灵按钮**（左对齐、单行**截断**）+ 右侧 `⋯` 菜单（重命名/删除）；选中态改**柔和苔绿**高亮（替代抢眼松绿实底）；
  4. 底部账户区：`管理后台`（admin）→ `👤 用户名 · 角色`（单行）+ `登出` → `ℹ️ 后端` 一行。
- **修复「⋯」拥挤**：列宽 `[6,1] → [0.8,0.2]`，并 CSS 收紧 popover 触发按钮内边距（不再压成「.·」）。
- **字体自适应**：侧栏 `h1/h2/h3` 覆盖为 Inter 无衬线（宋体仅留主区招牌/标题）；会话标题 `white-space:nowrap + text-overflow:ellipsis`；账户行同样 nowrap+省略，`管理员` 不再折行。
- **移除「清空当前对话内容」**：有历史后语义冗余；后端信息由 expander 收敛为一行 `st.caption`（保留「直接模式」文本供 E2E 与用户识别）。
- 会话列表包进 `st.container(key="convlist")` → DOM 带 `.st-key-convlist`，CSS 精确作用于会话行，不误伤其它按钮。

## 3. 验收
- SDD：新增 E2E `test_sidebar_structure_claude_like`——断言 `新建对话/最近对话/登出` 在位、`清空` 按钮计数为 0；随重整保持绿。
- 保留 E2E：会话切换、`⋯`→删除、快照重显、管理后台、登出等 14 项 E2E 全绿；66 单测全绿。
- 实景（Playwright）核验：`⋯` 菜单展开正常（重命名/删除），长标题截断，账户单行不折行，结构与 Claude 网页版一致。

## 4. 边界
- Streamlit 侧栏元素自上而下流式布局，账户区未做「绝对贴底」（会话多时随列表下移），以分隔线分区已足够清晰；如需贴底可后续用 flex `margin-top:auto`。
- 选中态高亮、幽灵按钮悬停等经注入 CSS 实现，随主题 token 统一。
